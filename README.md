# GEOточка: эскиз → DXF

MVP сервиса, который превращает фотографию геодезического эскиза в проверяемую геометрию и экспортирует её в DXF для AutoCAD.

> Сейчас это технологическая основа, а не готовая система распознавания. Детектор находит прямые линии, но размеры, подписи, проёмы и масштаб пользователь должен проверить перед экспортом.

## Что уже работает

- загрузка JPEG/PNG через API;
- базовая подготовка изображения и поиск прямых линий OpenCV;
- JSON с редактируемыми отрезками и оценкой уверенности;
- экспорт подтверждённой геометрии в ASCII DXF (AutoCAD R2010);
- автоматические тесты и проверка GitHub Actions.

## Быстрый запуск

Требуется Python 3.11+.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
uvicorn app.main:app --reload
```

Документация API будет доступна по адресу <http://127.0.0.1:8000/docs>.

## Пример работы

Распознать линии на фотографии:

```bash
curl -F 'file=@sketch.jpg' http://127.0.0.1:8000/api/v1/detect
```

После проверки координат экспортировать DXF:

```bash
curl -X POST http://127.0.0.1:8000/api/v1/export/dxf \
  -H 'Content-Type: application/json' \
  -d '{"units":"mm","scale_mm_per_pixel":10,"lines":[{"start":{"x":10,"y":20},"end":{"x":110,"y":20}}]}' \
  --output drawing.dxf
```

## Архитектура MVP

1. `image_processing.py` выравнивает контраст, выделяет границы и ищет отрезки.
2. API возвращает найденную геометрию, не скрывая неопределённость.
3. Клиент исправляет линии и задаёт масштаб по известному размеру.
4. `dxf_export.py` записывает подтверждённые отрезки на слой `WALLS`.

Следующие этапы: коррекция перспективы листа, OCR размеров, привязка чисел к размерным линиям, браузерный редактор и набор размеченных реальных эскизов.
